块级声明

块级声明用于声明在指定块作用域之外无法访问的变量。块级作用域(被称为词法作用域)存在于:

  • 函数内部
  • 块中({和}之间的区域)

let 声明

let声明的用法跟var相同，let的出现就是为了解决var变量提升这个痛点带来的麻烦。用let声明变量，就可以把变量的作用域限制在当前的代码块中。
由于let声明不会提升，因此开发者将let声明语句放在封闭代码块的顶部，以便整个代码块都可以访问。以下是示例：

const 与 let

const与let声明的都是块级标识符，所以常量也只能在当前代码块内有效，一旦执行到块外就会立刻被销毁。常量同样也不会提升至作用域顶部

与let相似，在同一作用域用const声明已经存在的标识符也会导致语法错误

用const声明对象

const声明的标识符不允许修改绑定，但允许修改值。这也就意味着用const声明对象之后，可以修改该对象的值

临时死区(temporal dead zone)或TDZ

JavaScript引擎在扫描代码的时候发现变量声明时，要么将它们提升至作用域顶部(遇到var声明)，要么将他们放到TDZ中(遇到let和const声明)
访问TDZ中的变量触发运行时的错误。只有在执行过变量声明之后，变量才会从TDZ中移出，然后才可以访问

循环中的函数

长久以来，var声明就让开发者在循环中创建函数变得异常困难，因为变量到了循环之外任能访问

你可能预期的结果是输出数字数字0~4，可是它却输出5次数字5
这是因为循环里的每次迭代同时共享着同一个变量i，循环内部创建的函数全部保留了对相同变量的引用。
循环结束时变量i的值为5，所以每次console.log(i)就会输出数字5

let声明模仿上述示例中IIFE所做的一切来简化循环过程。每次迭代循环都会创建一个新变量i，并将其初始化赋值为i的当前值
