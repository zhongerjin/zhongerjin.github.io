使用git遇到的问题

发表于 2019-03-25 | 分类于 技术

写这篇的原因

因为git是团队协作中最常使用的工具，除了正常操作还会发生一些幺蛾子事故。
为了下次遇到问题不再苦苦搜索答案，决定还是自己记录一下😂

关于Git操作

常用的有 git clone, git pull, git push, git merge, git rebase, git fetch, git commit, git add, git init，git reset

git clone

一般情况

如果想要克隆指定分支

git pull

将远程仓库的更改拉取到本地且合并

一般情况本地仓库适合远程仓库有关联的

本地已经有一个仓库，需要从远程仓库获取更改(两个仓库没任何联系)

首先要将本地仓库和远程仓库关联起来

origin就是远程仓库的名字，这是Git的默认叫法，也可以自定义

接下来执行git pull

这时候就会提示 refusing to merge unrelated histories after adding git remote
翻译过来就是远程仓库的历史记录跟本地仓库的历史记录不一致，git不允许合并

这时候只有两个选择

1.一个是克隆远程仓库，再将原来本地仓库的文件覆盖进去
2.添加选项忽略历史版本

第一个操作成本明显高很多，不容置疑肯定选择第二个
添加--allow-unrelated-histories即可

git push

一般情况

解惑

为什么有的时候会看到-u这个选项呢
使用-u就是将本地仓库和远程仓库关联起来
一般都是在第一次使用，以后push的时候就不用-u了

git reset

场景

提交更改之后发现代码时候有问题的，需要紧急撤回
查看历史记录以及版本号

先在本地回退

最后强制提交到远程仓库
